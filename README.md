# openclaw.telegram.mini.app

Проект **openclaw.telegram.mini.app** — демонстрация того, как можно собирать продукт и выстраивать инженерный процесс **исключительно силами OpenClaw**.

Этот репозиторий создаётся и развивается в рамках «Клуба незаменимых специалистов по искусственному интеллекту». Здесь работают два агента:

- **Front-end Developer** — пишет первую версию кода (лендинг/интерфейс Telegram Mini App).
- **Тестировщик (QA)** — делает ревью изменений и оставляет комментарии прямо в Pull Request.

## Как устроен флоу

- Все задачи, команды и согласования идут **через Telegram**.
- Весь проект и весь flow (ветки, коммиты, Pull Request, ревью, правки) создаётся и поддерживается **агентами**.

---

## Текущее содержимое (v1)

Первая версия — продающий лендинг (1 HTML‑страница) с технологичным AI‑дизайном под **Telegram Mini App**.

### Структура

- `index.html` — лендинг
- `assets/style.css` — стили
- `assets/app.js` — логика + интеграция Telegram WebApp

### Как открыть локально

Вариант 1 (самый простой):

```bash
cd /root/.openclaw/workspace/openclaw_fe_flow
python3 -m http.server 5173
```

Далее открыть:
- http://localhost:5173/

Вариант 2 (через Node):

```bash
npx serve .
```

### Telegram WebApp интеграция

В `index.html` подключён SDK:
- `https://telegram.org/js/telegram-web-app.js`

В `assets/app.js`:
- вызывается `Telegram.WebApp.ready()`
- используется `Telegram.WebApp.MainButton` как CTA (текст, show(), обработчик)
- минимально учитываются `themeParams` (bg/text/hint/link/button)
- есть fallback поведение вне Telegram

> Важно: в репозитории **нет** секретов/токенов.
